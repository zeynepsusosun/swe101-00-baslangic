Not Defteri CLI — Proje Raporu

TypeScript + Vitest ile geliştirilmiş komut satırı not defteri uygulaması

Geliştirici: Zeynep Su Sosun  ·  Tarih: 15 Temmuz 2026  ·  Proje: swe101-00-baslangic

Uygulama Özeti

Kullanıcıların komut satırından not ekleyebildiği, listeleyebildiği ve tamamlandı işaretleyebildiği küçük bir CLI uygulaması. Notlar notes.json dosyasında kalıcı olarak saklanır; uygulama kapatılsa bile notlar kaybolmaz.

Arayüz pastel pembe renk temasıyla (chalk kütüphanesi) renklendirilmiştir: tamamlanmış notlar üstü çizili mor, yüksek öncelikli notlar kırmızı, düşük öncelikli notlar yeşil görünür.

CLI Kullanımı

npm run dev -- ekle "Alışveriş yap"
npm run dev -- ekle "Makaleyi bitir" --oncelik yüksek
npm run dev -- ekle "Müzik listesi"  --oncelik düşük
npm run dev -- listele
npm run dev -- tamamla 1

Veri Modeli

type Priority = 'yüksek' | 'orta' | 'düşük'

interface Note {
  id:        number    // otomatik artan
  text:      string    // not içeriği
  done:      boolean   // tamamlanma durumu
  priority:  Priority  // öncelik seviyesi
  createdAt: string    // ISO 8601 tarihi
}

Mimari Kararlar

Neden JSON dosyası?

Veritabanı kurulumu gerektirmiyor, bağımlılık yok, dosya içeriği doğrudan okunabilir. Küçük ölçekli CLI araçları için yeterli.

Neden Dependency Injection?

addNote(text, priority, load, save) şeklinde load ve save fonksiyonları parametre olarak alınır. Testlerde gerçek notes.json'a dokunmadan in-memory kopya kullanılabilir — test izolasyonu sağlanmış olur.

Neden harici argüman-parser yok?

process.argv ile üç komut için yeterli. Komut sayısı artarsa yargs veya commander eklenebilir.

Ne Öğrendim

📁
JSON kalıcı depolama: fs.readFileSync / writeFileSync ile dosyadan okuma/yazma; existsSync ile ilk çalıştırmada dosya yokken güvenli başlangıç.
🔒
TypeScript strict modu: Dış verinin (JSON.parse) unknown tipinde geldiğini ve tip guard fonksiyonuyla doğrulanması gerektiğini öğrendim. any kullanmadan güvenli kod yazmak mümkün.
🔴
Test-first (TDD) döngüsü: Önce başarısız testi yazmak, ardından sadece o testi geçirecek kadar kod yazmak — fazladan soyutlamadan kaçınmayı kolaylaştırıyor.
🧪
Dependency injection ile test izolasyonu: loadNotes / saveNotes fonksiyonlarını parametre olarak geçirerek testlerde gerçek dosyaya dokunmadan in-memory kopya kullandım.
🎨
chalk ile terminal renklendirme: Hex renk kodlarıyla pastel tema oluşturma, strikethrough ile tamamlanmış not görünümü.
⌨️
CLI argüman ayrıştırma: process.argv ile harici kütüphane olmadan komut ve flag okuma.
📝
Anlamlı commit disiplini: Her özelliği ayrı commit'e bölmek, git log'u gelecekte okunabilir kılıyor ve her adımın ne yaptığını belgeliyor.
